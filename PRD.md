# Career Fit AI PRD

## 1. 产品定位

Career Fit AI 是一个面向中文实习/校招求职者的 AI 简历匹配助手。它让用户按维度填写求职画像，并判断该画像和目标岗位 JD 是否匹配，最后给出可执行的简历修改与面试准备建议。

第一阶段定位为作品集 MVP，重点展示 AI 产品思维、维度化判断逻辑和可运行能力。

## 2. 目标用户

- 正在寻找实习或校招机会的学生
- 有一段或多段实习经历，但不确定如何匹配 JD 的求职者
- 希望转向新方向的求职者，例如从招聘转 HRBP、从运营转产品等

## 3. 用户痛点

- 不知道自己是否适合投递某个岗位
- 看不懂 JD 中哪些要求是核心要求
- 自身背景和 JD 要求之间存在断层
- 不知道院校、专业、实习、项目、技能、岗位理解分别强弱在哪里
- 缺少具体的改写建议和面试准备方向
- 普通 ATS 分数不能解释中文实习场景下的实际匹配逻辑

## 4. MVP 范围

### 输入

- 分维度求职画像
- 岗位 JD 文本
- 可选：用户自带 OpenAI API key，用于深度分析

### 输出

- 总体匹配度
- 投递建议：强投 / 可投 / 谨慎投
- 总体判断
- 维度评分
- 每个维度的证据、缺口和建议
- 简历改写建议
- 面试准备问题
- 下一步行动

## 5. 评分维度

| 维度 | 权重 | 说明 |
| --- | ---: | --- |
| 学历/院校背景 | 15 | 自动识别学校是否具备 985/211/双一流标签，并将其作为竞争力参考，不作为单一判断依据。 |
| 专业相关性 | 10 | 判断专业背景是否能支撑岗位能力要求。 |
| 实习经历匹配 | 30 | 判断过往实习职责和 JD 的职责相似度。 |
| 项目成果与量化证据 | 20 | 判断是否有项目产出、数据指标、复盘或可验证结果。 |
| 技能工具匹配 | 15 | 判断工具、硬技能和通用能力是否匹配。 |
| 行业/岗位理解 | 10 | 判断是否体现对目标行业、业务线和岗位真实职责的理解。 |

## 6. 第一版不做什么

- 不做招聘方批量筛选
- 不做登录系统
- 不做数据库
- 不做 PDF/DOCX 上传
- 不自动编造简历经历
- 不输出无法解释的单一分数
- 不由项目作者承担公开访问场景下的 OpenAI API 调用费用
- 不保存用户 API key

## 7. 成功标准

- 用户可以在 1 分钟内完成一次 JD 与求职画像匹配分析
- 输出结果能指出具体背景证据，而不是泛泛评价
- 每个维度都有得分、证据、缺口和建议
- 改写建议必须基于用户已填写事实
- GitHub README 能清楚解释项目背景、功能和运行方式
